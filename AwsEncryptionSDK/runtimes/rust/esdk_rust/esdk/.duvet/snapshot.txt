SPECIFICATION: [Decrypt](specification/client-apis/decrypt.md)

    TEXT[!SHOULD,exception]: If an implementation requires holding the entire encrypted message in memory in order to perform this operation,

  SECTION: [Encrypted Message Format](#encrypted-message-format)

    TEXT[!SHOULD,exception]: that implementation SHOULD NOT provide an API that allows the caller to stream the encrypted message.

  SECTION: [Behavior](#behavior)
    TEXT[!MUST,implementation,test]: - Decrypt operation Step 1 MUST be [Parse the header](#parse-the-header)
    TEXT[!MUST,implementation,test]: - Decrypt operation Step 2 MUST be [Get the decryption materials](#get-the-decryption-materials)
    TEXT[!MUST,implementation,test]: - Decrypt operation Step 3 MUST be [Verify the header](#verify-the-header)
    TEXT[!MUST,implementation,test]: - Decrypt operation Step 4 MUST be [Decrypt the message body](#decrypt-the-message-body)
    TEXT[!MUST,implementation,test]: - Decrypt operation Step 5 MUST be [Verify the signature](#verify-the-signature)
    TEXT[!MUST,implementation,test]:   - If the message header contains an algorithm suite including a
    TEXT[!MUST,implementation,test]:     [signature algorithm](../framework/algorithm-suites.md#signature-algorithm),
    TEXT[!MUST,implementation,test]:     the Decrypt operation MUST perform this step.
    TEXT[!MUST,implementation,test]:   - If the message header does not contain an algorithm suite including a signature algorithm,
    TEXT[!MUST,implementation,test]:     the Decrypt operation MUST NOT perform this step.
    TEXT[!MUST,implementation,test]: If the input encrypted message is not being [streamed](streaming.md) to this operation,
    TEXT[!MUST,implementation,test]: all output MUST NOT be released until after these steps complete successfully.
    TEXT[!MUST,implementation,test]: - Output MUST NOT be released until otherwise indicated.
    TEXT[!MUST,implementation,test]: - If all bytes have been provided and this operation
    TEXT[!MUST,implementation,test]:   is unable to complete the above steps with the consumable encrypted message bytes,
    TEXT[!MUST,implementation,test]:   this operation MUST halt and indicate a failure to the caller.
    TEXT[!MUST,implementation,test]: - If this operation successfully completes the above steps
    TEXT[!MUST,implementation,test]:   but there are consumable bytes which are intended to be decrypted,
    TEXT[!MUST,implementation,test]:   this operation MUST fail.
    TEXT[!MUST,implementation,test]: - The ESDK MUST provide a configuration option that causes the decryption operation
    TEXT[!MUST,implementation,test]:   to fail immediately after parsing the header if a signed algorithm suite is used.

    TEXT[!MUST,implication]: successfully deserialized a valid [message header](../data-format/message-header.md).

    TEXT[!MUST,implementation,test]: or [Header Body Version 2.0](../data-format/message-header.md#header-body-version-20) specification,
    TEXT[!MUST,implementation,test]: depending on the [Version](../data-format/message-header.md#version) field.
    TEXT[!MUST,implementation,test]: Each header field MUST be deserialized according to its specification in the [message header](../data-format/message-header.md):
    TEXT[!MUST,implementation,test]: - [Version](../data-format/message-header.md#version): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Version](../data-format/message-header.md#version) specification.
    TEXT[!MUST,implementation,test]:   The value MUST be a [supported version](../data-format/message-header.md#supported-versions).
    TEXT[!MUST,implementation,test]: - [Type](../data-format/message-header.md#type) (V1 only): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Type](../data-format/message-header.md#type) specification.
    TEXT[!MUST,implementation,test]:   The value MUST be a [supported type](../data-format/message-header.md#supported-types).
    TEXT[!MUST,implementation,test]: - [Algorithm Suite ID](../data-format/message-header.md#algorithm-suite-id): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Algorithm Suite ID](../data-format/message-header.md#algorithm-suite-id) specification.
    TEXT[!MUST,implementation,test]: - [Message ID](../data-format/message-header.md#message-id): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Message ID](../data-format/message-header.md#message-id) specification.
    TEXT[!MUST,implementation,test]: - [AAD](../data-format/message-header.md#aad): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [AAD](../data-format/message-header.md#aad) specification.
    TEXT[!MUST,implementation,test]: - [Encrypted Data Keys](../data-format/message-header.md#encrypted-data-keys): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Encrypted Data Keys](../data-format/message-header.md#encrypted-data-keys) specification.
    TEXT[!MUST,implementation,test]: - [Content Type](../data-format/message-header.md#content-type): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Content Type](../data-format/message-header.md#content-type) specification.
    TEXT[!MUST,implementation,test]:   The value MUST be a [supported content type](../data-format/message-header.md#supported-content-types).
    TEXT[!MUST,implementation,test]: - [Reserved](../data-format/message-header.md#reserved) (V1 only): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Reserved](../data-format/message-header.md#reserved) specification.
    TEXT[!MUST,implementation,test]: - [IV Length](../data-format/message-header.md#iv-length) (V1 only): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [IV Length](../data-format/message-header.md#iv-length) specification.
    TEXT[!MUST,implementation,test]: - [Frame Length](../data-format/message-header.md#frame-length): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Frame Length](../data-format/message-header.md#frame-length) specification.
    TEXT[!MUST,implementation,test]: - [Algorithm Suite Data](../data-format/message-header.md#algorithm-suite-data) (V2 only): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Algorithm Suite Data](../data-format/message-header.md#algorithm-suite-data) specification.
    TEXT[!MUST,implementation,test]: - [Authentication Tag](../data-format/message-header.md#authentication-tag): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Authentication Tag](../data-format/message-header.md#authentication-tag) specification.
    TEXT[!MUST,implementation,test]: If the number of [encrypted data keys](../framework/structures.md#encrypted-data-keys)
    TEXT[!MUST,implementation,test]: deserialized from the [message header](../data-format/message-header.md)
    TEXT[!MUST,implementation,test]: is greater than the [maximum number of encrypted data keys](client.md#maximum-number-of-encrypted-data-keys) configured in the [client](client.md),
    TEXT[!MUST,implementation,test]: then as soon as that can be determined during deserializing
    TEXT[!MUST,implementation,test]: decrypt MUST process no more bytes and yield an error.
    TEXT[!MUST,implication]: This operation MUST wait if it doesn't have enough consumable encrypted message bytes to
    TEXT[!MUST,implication]: deserialize the next field of the message header until enough input bytes become consumable or
    TEXT[!MUST,implication]: the caller indicates an end to the encrypted message.
    TEXT[!MUST,implementation,test]: Until the [header is verified](#verify-the-header), this operation MUST NOT
    TEXT[!MUST,implementation,test]: release any parsed information from the header.

    TEXT[!MUST,implementation,test]: The CMM used MUST be the input CMM, if supplied.
    TEXT[!MUST,implementation,test]: If a CMM is not supplied as the input, the decrypt operation MUST construct a [default CMM](../framework/default-cmm.md)
    TEXT[!MUST,implementation,test]: from the input [keyring](../framework/keyring-interface.md).
    TEXT[!MUST,implementation,test]: The call to the CMM's [Decrypt Materials](../framework/cmm-interface.md#decrypt-materials) operation
    TEXT[!MUST,implementation,test]: MUST be constructed as follows:
    TEXT[!MUST,implementation,test]: - Encryption Context: This MUST be the parsed [encryption context](../data-format/message-header.md#aad)
    TEXT[!MUST,implementation,test]:   from the message header.
    TEXT[!MUST,implementation,test]: - Algorithm Suite ID: This MUST be the parsed
    TEXT[!MUST,implementation,test]:   [algorithm suite ID](../data-format/message-header.md#algorithm-suite-id)
    TEXT[!MUST,implementation,test]:   from the message header.
    TEXT[!MUST,implementation,test]: - Encrypted Data Keys: This MUST be the parsed [encrypted data keys](../data-format/message-header#encrypted-data-keys)

    TEXT[!MUST,implementation,test]: - Commitment Policy: This MUST be the commitment policy configured on the client.
    TEXT[!MUST,implementation,test]: The data key used as input for all decryption described below MUST be a data key derived from the plaintext data key

    TEXT[!MUST,implementation,test]: encrypt MUST yield an error.
    TEXT[!MUST,implementation,test]: If the algorithm suite is not supported by the [commitment policy](client.md#commitment-policy)
    TEXT[!MUST,implementation,test]: configured in the [client](client.md) decrypt MUST yield an error.
    TEXT[!MUST,implementation,test]: If the [algorithm suite](../framework/algorithm-suites.md#algorithm-suites-encryption-key-derivation-settings) supports [key commitment](../framework/algorithm-suites.md#key-commitment)
    TEXT[!MUST,implementation,test]: then the [commit key](../framework/algorithm-suites.md#commit-key) MUST be derived from the plaintext data key
    TEXT[!MUST,implementation,test]: using the [commit key derivation](../framework/algorithm-suites.md#algorithm-suites-commit-key-derivation-settings).
    TEXT[!MUST,implementation,test]: The derived commit key MUST equal the commit key stored in the message header.
    TEXT[!MUST,implementation,test]: The algorithm suite used to derive a data key from the plaintext data key MUST be
    TEXT[!MUST,implementation,test]: the [key derivation algorithm](../framework/algorithm-suites.md#key-derivation-algorithm) included in the
    TEXT[!MUST,implementation,test]: [algorithm suite](../framework/algorithm-suites.md) associated with
    TEXT[!MUST,implementation,test]: the returned decryption materials.
    TEXT[!MUST,implementation,test]: If the key derivation algorithm is the [identity KDF](../framework/algorithm-suites.md#identity-kdf),
    TEXT[!MUST,implementation,test]: then the derived data key MUST be the same as the plaintext data key.

  SECTION: [Verify the header](#verify-the-header)
    TEXT[!MUST,implementation,test]: Once a valid message header is deserialized and decryption materials are available,
    TEXT[!MUST,implementation,test]: this operation MUST validate the [message header body](../data-format/message-header.md#header-body)
    TEXT[!MUST,implementation,test]: by using the [authenticated encryption algorithm](../framework/algorithm-suites.md#encryption-algorithm)
    TEXT[!MUST,implementation,test]: to decrypt with the following inputs:
    TEXT[!MUST,implementation,test]: - The AAD MUST be the concatenation of the serialized [message header body](../data-format/message-header.md#header-body)
    TEXT[!MUST,implementation,test]:   and the serialization of encryption context to only authenticate.
    TEXT[!MUST,implementation,test]:   The encryption context to only authenticate MUST be the [encryption context](../framework/structures.md#encryption-context)
    TEXT[!MUST,implementation,test]:   in the [decryption materials](../framework/structures.md#decryption-materials)
    TEXT[!MUST,implementation,test]:   filtered to only contain key value pairs listed in
    TEXT[!MUST,implementation,test]:   the [decryption material's](../framework/structures.md#decryption-materials)
    TEXT[!MUST,implementation,test]:   [required encryption context keys](../framework/structures.md#required-encryption-context-keys-1)
    TEXT[!MUST,implementation,test]:   serialized according to the [encryption context serialization specification](../framework/structures.md#serialization).
    TEXT[!MUST,implementation,test]: - For message format version [1.0](../data-format/message-header.md#supported-versions)
    TEXT[!MUST,implementation,test]:   the IV MUST be the value serialized in the message header's [IV field](../data-format/message-header#iv).
    TEXT[!MUST,implementation,test]:   For message format version [2.0](../data-format/message-header.md#supported-versions)
    TEXT[!MUST,implementation,test]:   the IV MUST be 0.
    TEXT[!MUST,implementation,test]: - the cipherkey MUST be the derived data key
    TEXT[!MUST,implementation,test]: - the ciphertext MUST be an empty byte array
    TEXT[!MUST,implementation,test]: - the tag MUST be the value serialized in the message header's
    TEXT[!MUST,implementation,test]:   [authentication tag field](../data-format/message-header.md#authentication-tag)
    TEXT[!MUST,implementation,test]: If this tag verification fails, this operation MUST immediately halt and fail.
    TEXT[!SHOULD,exception]: - A streamed Decrypt operation SHOULD release the parsed [encryption context](#encryption-context),
    TEXT[!SHOULD,exception]:   [algorithm suite ID](../data-format/message-header.md#algorithm-suite-id),
    TEXT[!SHOULD,exception]:   and [other header information](#parsed-header)
    TEXT[!SHOULD,exception]:   as soon as tag verification succeeds.
    TEXT[!MUST,implementation,test]:   However, if the streamed Decrypt operation is using an algorithm suite with a signature algorithm
    TEXT[!MUST,implementation,test]:   all released output MUST NOT be considered signed data until
    TEXT[!MUST,implementation,test]:   this operation successfully completes.
    TEXT[!SHOULD,implication]: - The streamed Decrypt operation SHOULD input the serialized header to the signature algorithm as soon as it is deserialized,
    TEXT[!SHOULD,implication]:   such that the serialized header isn't required to remain in memory to [verify the signature](#verify-the-signature).

  SECTION: [Decrypt the message body](#decrypt-the-message-body)

    TEXT[!MUST,implementation,test]: Non-framed data deserialization MUST conform to the [Non-Framed Data](../data-format/message-body.md#non-framed-data) specification.
    TEXT[!MUST,implementation,test]: Once the message header is successfully parsed, the next sequential bytes
    TEXT[!MUST,implementation,test]: MUST be deserialized according to the [message body spec](../data-format/message-body.md).
    TEXT[!MUST,implementation,test]: If there could still be message body left to deserialize and decrypt,
    TEXT[!MUST,implementation,test]: this operation MUST either wait for more of the encrypted message bytes to become consumable,
    TEXT[!MUST,implementation,test]: wait for the end to the encrypted message to be indicated,
    TEXT[!MUST,implementation,test]: or deserialize and/or decrypt the consumable bytes.
    TEXT[!MUST,implementation,test]: The Decrypt operation MUST use the [content type](../data-format/message-header.md#content-type) field parsed from the
    TEXT[!MUST,implementation,test]: message header to determine whether the operation will deserialize the message bytes as
    TEXT[!MUST,implementation,test]: [framed data](../data-format/message-body.md#framed-data) or
    TEXT[!MUST,implementation,test]: [un-framed data](../data-format/message-body.md#non-framed-data).
    TEXT[!MUST,implementation,test]: If deserializing [framed data](../data-format/message-body.md#framed-data),
    TEXT[!MUST,implementation,test]: the Decrypt operation MUST use the first 4 bytes of a frame to determine
    TEXT[!MUST,implementation,test]: whether the operation will deserialize the frame as a [final frame](../data-format/message-body.md#final-frame)
    TEXT[!MUST,implementation,test]: or [regular frame](../data-format/message-body.md#regular-frame).
    TEXT[!MUST,implementation,test]: - The [Sequence Number End](../data-format/message-body.md#sequence-number-end): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Sequence Number End](../data-format/message-body.md#sequence-number-end) specification.
    TEXT[!MUST,implementation,test]:   If the first 4 bytes have a value of 0xFFFF,
    TEXT[!MUST,implementation,test]:   then the Decrypt operation MUST deserialize the following bytes according to the [final frame spec](../data-format/message-body.md#final-frame).
    TEXT[!MUST,implementation,test]:   Otherwise, the Decrypt operation MUST deserialize the bytes according to the [regular frame spec](../data-format/message-body.md#regular-frame).
    TEXT[!MUST,implementation,test]: - [Sequence Number](../data-format/message-body.md#regular-frame-sequence-number): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Regular Frame Sequence Number](../data-format/message-body.md#regular-frame-sequence-number) specification.
    TEXT[!MUST,implementation,test]: - [IV](../data-format/message-body.md#regular-frame-iv): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Regular Frame IV](../data-format/message-body.md#regular-frame-iv) specification.
    TEXT[!MUST,implementation,test]: - [Encrypted Content Length](../data-format/message-body.md#final-frame-encrypted-content-length): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Final Frame Encrypted Content Length](../data-format/message-body.md#final-frame-encrypted-content-length) specification.
    TEXT[!MUST,implementation,test]:   If deserializing a [final frame](../data-format/message-body.md#final-frame),
    TEXT[!MUST,implementation,test]:   the Decrypt operation MUST ensure that the length of the encrypted content field is
    TEXT[!MUST,implementation,test]:   less than or equal to the frame length deserialized in the message header.
    TEXT[!MUST,implementation,test]: - [Encrypted Content](../data-format/message-body.md#regular-frame-encrypted-content): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Regular Frame Encrypted Content](../data-format/message-body.md#regular-frame-encrypted-content) specification.
    TEXT[!MUST,implementation,test]: - [Authentication Tag](../data-format/message-body.md#regular-frame-authentication-tag): MUST be deserialized according to the
    TEXT[!MUST,implementation,test]:   [Regular Frame Authentication Tag](../data-format/message-body.md#regular-frame-authentication-tag) specification.
    TEXT[!MUST,implementation,test]: Once at least a single frame is deserialized (or the entire body in the un-framed case),
    TEXT[!MUST,implementation,test]: the Decrypt operation MUST decrypt and authenticate the frame (or body) using the
    TEXT[!MUST,implementation,test]: [authenticated encryption algorithm](../framework/algorithm-suites.md#encryption-algorithm)
    TEXT[!MUST,implementation,test]: specified by the [algorithm suite](../framework/algorithm-suites.md), with the following inputs:
    TEXT[!MUST,implementation,test]: - The AAD MUST be the serialized [message body AAD](../data-format/message-body-aad.md),
    TEXT[!MUST,implementation,test]:   constructed according to the [Message Body AAD](../data-format/message-body-aad.md) specification, as follows:
    TEXT[!MUST,implementation,test]:   - The [message ID](../data-format/message-body-aad.md#message-id) MUST be the same as the

    TEXT[!MUST,implementation,test]:     according to [Message Body AAD](../data-format/message-body-aad.md) depending on
    TEXT[!MUST,implementation,test]:     whether the bytes being decrypted are a regular frame, final frame, or un-framed data.
    TEXT[!MUST,implementation,test]:   - The [sequence number](../data-format/message-body-aad.md#sequence-number) MUST be the sequence
    TEXT[!MUST,implementation,test]:     number deserialized from the frame being decrypted.
