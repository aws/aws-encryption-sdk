SPECIFICATION: [Decrypt](specification/client-apis/decrypt.md)
  SECTION: [Overview](#overview)
    TEXT[!MUST]: Any client provided by the AWS Encryption SDK that performs decryption of encrypted messages MUST follow
    TEXT[!MUST]: this specification for decryption.

  SECTION: [Authenticated Data](#authenticated-data)
    TEXT[!MUST]: This operation MUST NOT release any unauthenticated plaintext or unauthenticated associated data.

  SECTION: [Input](#input)
    TEXT[!MUST,implementation]: - The input to the Decrypt operation MUST accept a required [Encrypted Message](#encrypted-message) argument.
    TEXT[!MUST,implementation]: - The input to the Decrypt operation MUST accept a [cryptographic Materials Manager (CMM)](../framework/cmm-interface.md) and a [keyring](../framework/keyring-interface.md) argument.
    TEXT[!SHOULD]:   The keyring and CMM inputs SHOULD be optional.
    TEXT[!MUST]:   The Decrypt operation MUST validate that exactly one keyring or CMM was provided by the caller.
    TEXT[!MUST]:   If the caller does not provide exactly one of a keyring or CMM, the Decrypt operation MUST fail.
    TEXT[!MUST,implementation]: - The input to the Encrypt operation MUST accept an optional [Encryption Context](#encryption-context) argument.

  SECTION: [Encrypted Message](#encrypted-message)
    TEXT[!MUST]: The input encrypted message MUST be a sequence of bytes in the
    TEXT[!MUST]: [message format](../data-format/message.md) specified by the AWS Encryption SDK.
    TEXT[!MAY]: This input MAY be [streamed](streaming.md) to this operation.
    TEXT[!SHOULD]: If an implementation requires holding the entire encrypted message in memory in order to perform this operation,
    TEXT[!SHOULD]: that implementation SHOULD NOT provide an API that allows the caller to stream the encrypted message.

  SECTION: [Encrypted Message Format](#encrypted-message-format)
    TEXT[!SHOULD]: To make diagnosing this mistake easier, implementations SHOULD detect the first two bytes of the Base64 encoding of any supported message [versions](../data-format/message-header.md#version-1)
    TEXT[!SHOULD]: and [types](../data-format/message-header.md#type)
    TEXT[!SHOULD]: and fail with a more specific error message.

  SECTION: [Cryptographic Materials Manager](#cryptographic-materials-manager)
    TEXT[!MUST]: This CMM MUST obtain the [decryption materials](../framework/structures.md#decryption-materials) required for decryption.

  SECTION: [Keyring](#keyring)
    TEXT[!MUST]: If the Keyring is provided as the input, the client MUST construct a [default CMM](../framework/default-cmm.md) that uses this keyring,
    TEXT[!MUST]: to obtain the [decryption materials](../framework/structures.md#decryption-materials) that is required for decryption.
    TEXT[!MUST]: This default CMM MUST obtain the decryption materials required for decryption.

  SECTION: [Output](#output)
    TEXT[!MUST,implementation]: - The output of the Decrypt operation MUST include a [Plaintext](#plaintext) value.
    TEXT[!MUST,implementation]: - The output of the Decrypt operation MUST include an [encryption context](#encryption-context) value.
    TEXT[!MUST,implementation]: - The output of the Decrypt operation MUST include an [algorithm suite](#algorithm-suite) value.
    TEXT[!SHOULD,exception]: - The output of the Decrypt operation SHOULD include a [Parsed Header](#parsed-header) value.

  SECTION: [Plaintext](#plaintext)
    TEXT[!MAY]: This operation MAY [stream](streaming.md) the plaintext as output.
    TEXT[!SHOULD]: If an implementation requires holding the entire encrypted message in memory in order to perform this operation,
    TEXT[!SHOULD]: that implementation SHOULD NOT provide an API that allows the caller to stream the encrypted message.

  SECTION: [Encryption Context](#encryption-context)
    TEXT[!MAY]: This output MAY be satisfied by outputting a [parsed header](#parsed-header) containing this value.

  SECTION: [Algorithm Suite](#algorithm-suite)
    TEXT[!MUST]: This algorithm suite MUST be [supported for the ESDK](../framework/algorithm-suites.md#supported-algorithm-suites-enum).
    TEXT[!MAY]: This output MAY be satisfied by outputting a [parsed header](#parsed-header) containing this value.

  SECTION: [Behavior](#behavior)
    TEXT[!MUST,implementation]: - Decrypt operation Step 1 MUST be [Parse the header](#parse-the-header)
    TEXT[!MUST]: - Decrypt operation Step 2 MUST be [Get the decryption materials](#get-the-decryption-materials)
    TEXT[!MUST]: - Decrypt operation Step 3 MUST be [Verify the header](#verify-the-header)
    TEXT[!MUST]: - Decrypt operation Step 4 MUST be [Decrypt the message body](#decrypt-the-message-body)
    TEXT[!MUST]: - Decrypt operation Step 5 MUST be [Verify the signature](#verify-the-signature)
    TEXT[!MUST]:   - If the message header contains an algorithm suite including a
    TEXT[!MUST]:     [signature algorithm](../framework/algorithm-suites.md#signature-algorithm),
    TEXT[!MUST]:     the Decrypt operation MUST perform this step.
    TEXT[!MUST]:   - If the message header does not contain an algorithm suite including a signature algorithm,
    TEXT[!MUST]:     the Decrypt operation MUST NOT perform this step.
    TEXT[!MUST,implementation]: If the input encrypted message is not being [streamed](streaming.md) to this operation,
    TEXT[!MUST,implementation]: all output MUST NOT be released until after these steps complete successfully.
    TEXT[!MUST]: - Output MUST NOT be released until otherwise indicated.
    TEXT[!MUST]: - If all bytes have been provided and this operation
    TEXT[!MUST]:   is unable to complete the above steps with the consumable encrypted message bytes,
    TEXT[!MUST]:   this operation MUST halt and indicate a failure to the caller.
    TEXT[!MUST,implementation]: - If this operation successfully completes the above steps
    TEXT[!MUST,implementation]:   but there are consumable bytes which are intended to be decrypted,
    TEXT[!MUST,implementation]:   this operation MUST fail.
    TEXT[!MUST]: - The ESDK MUST provide a configuration option that causes the decryption operation
    TEXT[!MUST]:   to fail immediately after parsing the header if a signed algorithm suite is used.

  SECTION: [Parse the header](#parse-the-header)
    TEXT[!MUST,implementation]: Given encrypted message bytes, this operation MUST process those bytes sequentially,
    TEXT[!MUST,implementation]: deserializing those bytes according to the [message format](../data-format/message.md).
    TEXT[!MUST]: This operation MUST attempt to deserialize all consumable encrypted message bytes until it has
    TEXT[!MUST]: successfully deserialized a valid [message header](../data-format/message-header.md).
    TEXT[!MUST,implementation]: If the number of [encrypted data keys](../framework/structures.md#encrypted-data-keys)
    TEXT[!MUST,implementation]: deserialized from the [message header](../data-format/message-header.md)
    TEXT[!MUST,implementation]: is greater than the [maximum number of encrypted data keys](client.md#maximum-number-of-encrypted-data-keys) configured in the [client](client.md),
    TEXT[!MUST,implementation]: then as soon as that can be determined during deserializing
    TEXT[!MUST,implementation]: decrypt MUST process no more bytes and yield an error.
    TEXT[!MUST]: This operation MUST wait if it doesn't have enough consumable encrypted message bytes to
    TEXT[!MUST]: deserialize the next field of the message header until enough input bytes become consumable or
    TEXT[!MUST]: the caller indicates an end to the encrypted message.
    TEXT[!MUST]: Until the [header is verified](#verify-the-header), this operation MUST NOT
    TEXT[!MUST]: release any parsed information from the header.

  SECTION: [Get the decryption materials](#get-the-decryption-materials)
    TEXT[!MUST,implementation]: If the parsed [algorithm suite ID](../data-format/message-header.md#algorithm-suite-id)
    TEXT[!MUST,implementation]: is not supported by the [commitment policy](client.md#commitment-policy)
    TEXT[!MUST,implementation]: configured in the [client](client.md) decrypt MUST yield an error.
    TEXT[!MUST,implementation]: This operation MUST obtain this set of [decryption materials](../framework/structures.md#decryption-materials),
    TEXT[!MUST,implementation]: by calling [Decrypt Materials](../framework/cmm-interface.md#decrypt-materials) on a [CMM](../framework/cmm-interface.md).
    TEXT[!MUST,implementation]: The CMM used MUST be the input CMM, if supplied.
    TEXT[!MUST,implementation]: If a CMM is not supplied as the input, the decrypt operation MUST construct a [default CMM](../framework/default-cmm.md)
    TEXT[!MUST,implementation]: from the input [keyring](../framework/keyring-interface.md).
    TEXT[!MUST,implementation]: The call to the CMM's [Decrypt Materials](../framework/cmm-interface.md#decrypt-materials) operation
    TEXT[!MUST,implementation]: MUST be constructed as follows:
    TEXT[!MUST]: - Encryption Context: This MUST be the parsed [encryption context](../data-format/message-header.md#aad)
    TEXT[!MUST]:   from the message header.
    TEXT[!MUST]: - Algorithm Suite ID: This MUST be the parsed
    TEXT[!MUST]:   [algorithm suite ID](../data-format/message-header.md#algorithm-suite-id)
    TEXT[!MUST]:   from the message header.
    TEXT[!MUST]: - Encrypted Data Keys: This MUST be the parsed [encrypted data keys](../data-format/message-header#encrypted-data-keys)
    TEXT[!MUST]:   from the message header.
    TEXT[!MUST]: - Reproduced Encryption Context: This MUST be the [input](#input) encryption context.
    TEXT[!MUST,implementation]: The data key used as input for all decryption described below MUST be a data key derived from the plaintext data key
    TEXT[!MUST,implementation]: included in the [decryption materials](../framework/structures.md#decryption-materials).
    TEXT[!MUST,implementation]: The algorithm suite used as input for all decryption described below MUST be the algorithm suite
    TEXT[!MUST,implementation]: included in the [decryption materials](../framework/structures.md#decryption-materials).
    TEXT[!MUST]: If this algorithm suite is not [supported for the ESDK](../framework/algorithm-suites.md#supported-algorithm-suites-enum)
    TEXT[!MUST]: encrypt MUST yield an error.
    TEXT[!MUST]: If the algorithm suite is not supported by the [commitment policy](client.md#commitment-policy)
    TEXT[!MUST]: configured in the [client](client.md) decrypt MUST yield an error.
    TEXT[!MUST,implementation]: If the [algorithm suite](../framework/algorithm-suites.md#algorithm-suites-encryption-key-derivation-settings) supports [key commitment](../framework/algorithm-suites.md#key-commitment)
    TEXT[!MUST,implementation]: then the [commit key](../framework/algorithm-suites.md#commit-key) MUST be derived from the plaintext data key
    TEXT[!MUST,implementation]: using the [commit key derivation](../framework/algorithm-suites.md#algorithm-suites-commit-key-derivation-settings).
    TEXT[!MUST,implementation]: The derived commit key MUST equal the commit key stored in the message header.
    TEXT[!MUST,implementation]: The algorithm suite used to derive a data key from the plaintext data key MUST be
    TEXT[!MUST,implementation]: the [key derivation algorithm](../framework/algorithm-suites.md#key-derivation-algorithm) included in the
    TEXT[!MUST,implementation]: [algorithm suite](../framework/algorithm-suites.md) associated with
    TEXT[!MUST,implementation]: the returned decryption materials.
    TEXT[!MUST]: If the key derivation algorithm is the [identity KDF](../framework/algorithm-suites.md#identity-kdf),
    TEXT[!MUST]: then the derived data key MUST be the same as the plaintext data key.

  SECTION: [Verify the header](#verify-the-header)
    TEXT[!MUST,implementation]: Once a valid message header is deserialized and decryption materials are available,
    TEXT[!MUST,implementation]: this operation MUST validate the [message header body](../data-format/message-header.md#header-body)
    TEXT[!MUST,implementation]: by using the [authenticated encryption algorithm](../framework/algorithm-suites.md#encryption-algorithm)
    TEXT[!MUST,implementation]: to decrypt with the following inputs:
    TEXT[!MUST,implementation]: - The AAD MUST be the concatenation of the serialized [message header body](../data-format/message-header.md#header-body)
    TEXT[!MUST,implementation]:   and the serialization of encryption context to only authenticate.
    TEXT[!MUST,implementation]:   The encryption context to only authenticate MUST be the [encryption context](../framework/structures.md#encryption-context)
    TEXT[!MUST,implementation]:   in the [decryption materials](../framework/structures.md#decryption-materials)
    TEXT[!MUST,implementation]:   filtered to only contain key value pairs listed in
    TEXT[!MUST,implementation]:   the [decryption material's](../framework/structures.md#decryption-materials)
    TEXT[!MUST,implementation]:   [required encryption context keys](../framework/structures.md#required-encryption-context-keys-1)
    TEXT[!MUST,implementation]:   serialized according to the [encryption context serialization specification](../framework/structures.md#serialization).
    TEXT[!MUST]: - For message format version [1.0](../data-format/message-header.md#supported-versions)
    TEXT[!MUST]:   the IV MUST be the value serialized in the message header's [IV field](../data-format/message-header#iv).
    TEXT[!MUST]:   For message format version [2.0](../data-format/message-header.md#supported-versions)
    TEXT[!MUST]:   the IV MUST be 0.
    TEXT[!MUST]: - the cipherkey MUST be the derived data key
    TEXT[!MUST]: - the ciphertext MUST be an empty byte array
    TEXT[!MUST]: - the tag MUST be the value serialized in the message header's
    TEXT[!MUST]:   [authentication tag field](../data-format/message-header.md#authentication-tag)
    TEXT[!MUST,implementation]: If this tag verification fails, this operation MUST immediately halt and fail.
    TEXT[!SHOULD]: - A streamed Decrypt operation SHOULD release the parsed [encryption context](#encryption-context),
    TEXT[!SHOULD]:   [algorithm suite ID](../data-format/message-header.md#algorithm-suite-id),
    TEXT[!SHOULD]:   and [other header information](#parsed-header)
    TEXT[!SHOULD]:   as soon as tag verification succeeds.
    TEXT[!MUST]:   However, if the streamed Decrypt operation is using an algorithm suite with a signature algorithm
    TEXT[!MUST]:   all released output MUST NOT be considered signed data until
    TEXT[!MUST]:   this operation successfully completes.
    TEXT[!SHOULD]: - The streamed Decrypt operation SHOULD input the serialized header to the signature algorithm as soon as it is deserialized,
    TEXT[!SHOULD]:   such that the serialized frame isn't required to remain in memory to [verify the signature](#verify-the-signature).

  SECTION: [Decrypt the message body](#decrypt-the-message-body)
    TEXT[!MUST,implementation]: Once the message header is successfully parsed, the next sequential bytes
    TEXT[!MUST,implementation]: MUST be deserialized according to the [message body spec](../data-format/message-body.md).
    TEXT[!MUST]: If there could still be message body left to deserialize and decrypt,
    TEXT[!MUST]: this operation MUST either wait for more of the encrypted message bytes to become consumable,
    TEXT[!MUST]: wait for the end to the encrypted message to be indicated,
    TEXT[!MUST]: or deserialize and/or decrypt the consumable bytes.
    TEXT[!MUST,implementation]: The Decrypt operation MUST use the [content type](../data-format/message-header.md#content-type) field parsed from the
    TEXT[!MUST,implementation]: message header to determine whether the operation will deserialize the message bytes as
    TEXT[!MUST,implementation]: [framed data](../data-format/message-body.md#framed-data) or
    TEXT[!MUST,implementation]: [un-framed data](../data-format/message-body.md#un-framed-data).
    TEXT[!MUST,implementation]: If deserializing [framed data](../data-format/message-body.md#framed-data),
    TEXT[!MUST,implementation]: the Decrypt operation operation MUST use the first 4 bytes of a frame to determine
    TEXT[!MUST,implementation]: whether the operation will deserialize the frame as a [final frame](../data-format/message-body.md#final-frame)
    TEXT[!MUST,implementation]: or [regular frame](../fata-format/message-body/md#regular-frame).
    TEXT[!MUST,implementation]: If the first 4 bytes have a value of 0xFFFF,
    TEXT[!MUST,implementation]: then the Decrypt operation MUST deserialize this as the [sequence number end](../data-format/message-header.md#sequence-number-end)
    TEXT[!MUST,implementation]: and the following bytes according to the [final frame spec](../data-format/message-body.md#final-frame).
    TEXT[!MUST,implementation]: Otherwise, the Decrypt operation MUST deserialize this as the [sequence number](../data-format/message-header.md#sequence-number)
    TEXT[!MUST,implementation]: and the following bytes according to the [regular frame spec](../data-format/message-body.md#regular-frame).
    TEXT[!MUST,implementation]: If deserializing a [final frame](../data-format/message-body.md#final-frame),
    TEXT[!MUST,implementation]: the Decrypt operation MUST ensure that the length of the encrypted content field is
    TEXT[!MUST,implementation]: less than or equal to the frame length deserialized in the message header.
    TEXT[!MUST,implementation]: Once at least a single frame is deserialized (or the entire body in the un-framed case),
    TEXT[!MUST,implementation]: the Decrypt operation MUST decrypt and authenticate the frame (or body) using the
    TEXT[!MUST,implementation]: [authenticated encryption algorithm](../framework/algorithm-suites.md#encryption-algorithm)
    TEXT[!MUST,implementation]: specified by the [algorithm suite](../framework/algorithm-suites.md), with the following inputs:
    TEXT[!MUST]: - The AAD MUST be the serialized [message body AAD](../data-format/message-body-aad.md),
    TEXT[!MUST]:   constructed as follows:
    TEXT[!MUST]:   - The [message ID](../data-format/message-body-aad.md#message-id) MUST be the same as the
    TEXT[!MUST]:     [message ID](../data-frame/message-header.md#message-id) deserialized from the header of this message.
    TEXT[!MUST]:   - The [Body AAD Content](../data-format/message-body-aad.md#body-aad-content) MUST be constructed
    TEXT[!MUST]:     according to [Message Body AAD](../data-format/message-body-aad.md) depending on
    TEXT[!MUST]:     whether the bytes being decrypted are a regular frame, final frame, or un-framed data.
    TEXT[!MUST]:   - The [sequence number](../data-format/message-body-aad.md#sequence-number) MUST be the sequence
    TEXT[!MUST]:     number deserialized from the frame being decrypted.
    TEXT[!MUST]:     If this is un-framed data, this value MUST be 1.
    TEXT[!MUST,implementation]:     If this is framed data and the first frame sequentially, this value MUST be 1.
    TEXT[!MUST,implementation]:     Otherwise, this value MUST be 1 greater than the value of the sequence number
    TEXT[!MUST,implementation]:     of the previous frame.
    TEXT[!MUST,implementation]:   - The [content length](../data-format/message-body-aad.md#content-length) MUST have a value
    TEXT[!MUST,implementation]:     equal to the length of the plaintext that was encrypted.
    TEXT[!SHOULD]:     If this is a regular frame, this SHOULD be determined by using the [frame length](../data-format/message-header.md#frame-length)
    TEXT[!SHOULD]:     deserialized from the message header.
    TEXT[!SHOULD]:     If this is not a regular frame, this SHOULD be determined by using the the [encrypted content length](../data-format/message-body.md#encrypted-content-length).
    TEXT[!MUST]: - The IV MUST be the [sequence number](../data-format/message-body-aad.md#sequence-number)
    TEXT[!MUST]:   used in the message body AAD above,
    TEXT[!MUST]:   padded to the [IV length](../data-format/message-header.md#iv-length) with 0.
    TEXT[!MUST]: - The cipherkey MUST be the derived data key
    TEXT[!MUST]: - The ciphertext MUST be the [encrypted content](../data-format/message-body.md#encrypted-content).
    TEXT[!MUST]: - The tag MUST be the value serialized in the
    TEXT[!MUST]:   [authentication tag field](../data-format/message-body.md#authentication-tag)
    TEXT[!MUST]:   in the message body or frame.
    TEXT[!MUST,implementation]: If this decryption fails, this operation MUST immediately halt and fail.
    TEXT[!MUST,implementation]: This operation MUST NOT release any unauthenticated plaintext.
    TEXT[!SHOULD]: - If the streamed Decrypt operation is using an algorithm suite without a signature algorithm,
    TEXT[!SHOULD]:   plaintext SHOULD be released as soon as the above calculation, including tag verification,
    TEXT[!SHOULD]:   succeeds.
    TEXT[!SHOULD,implementation]: - If the streamed Decrypt operation is using an algorithm suite with a signature algorithm,
    TEXT[!SHOULD,implementation]:   all plaintext decrypted from regular frames SHOULD be released as soon as the above calculation,
    TEXT[!SHOULD,implementation]:   including tag verification, succeeds.
    TEXT[!MUST,implementation]:   Any plaintext decrypted from [unframed data](../data-format/message-body.md#un-framed-data) or
    TEXT[!MUST,implementation]:   a final frame in a streamed Decrypt operation MUST NOT be released until [signature verification](#verify-the-signature)
    TEXT[!MUST,implementation]:   successfully completes.
    TEXT[!SHOULD]: - The streamed Decrypt operation SHOULD input the serialized frame to the signature algorithm as soon as it is deserialized,
    TEXT[!SHOULD]:   such that the serialized frame isn't required to remain in memory to complete
    TEXT[!SHOULD]:   the [signature verification](#verify-the-signature).

  SECTION: [Verify the signature](#verify-the-signature)
    TEXT[!MUST,implementation]: If the algorithm suite has a signature algorithm,
    TEXT[!MUST,implementation]: the Decrypt operation MUST verify the message footer using the specified signature algorithm.
    TEXT[!MUST,implementation]: After deserializing the body, the Decrypt operation MUST deserialize the next encrypted message bytes
    TEXT[!MUST,implementation]: as the [message footer](../data-format/message-footer.md).
    TEXT[!MUST]: If there are not enough consumable bytes to deserialize the message footer and
    TEXT[!MUST]: the caller has not yet indicated an end to the encrypted message,
    TEXT[!MUST]: the Decrypt operation MUST wait for enough bytes to become consumable or for the caller
    TEXT[!MUST]: to indicate an end to the encrypted message.
    TEXT[!MUST,implementation]: Once the message footer is deserialized, the Decrypt operation MUST use the
    TEXT[!MUST,implementation]: [signature algorithm](../framework/algorithm-suites.md#signature-algorithm)
    TEXT[!MUST,implementation]: from the [algorithm suite](../framework/algorithm-suites.md) in the decryption materials to
    TEXT[!MUST,implementation]: verify the encrypted message, with the following inputs:
    TEXT[!MUST]: - The verification key MUST be the [verification key](../framework/structures.md#verification-key)
    TEXT[!MUST]:   in the decryption materials.
    TEXT[!MUST]: - The input to verify MUST be the concatenation of the serialization of the
    TEXT[!MUST]:   [message header](../data-format/message-header.md) and [message body](../data-format/message-body.md).
    TEXT[!MUST,implementation]: If this verification is not successful, this operation MUST immediately halt and fail.

  SECTION: [Security Considerations](#security-considerations)
    TEXT[!MUST]: If this operation is [streaming](streaming.md) output to the caller
    TEXT[!MUST]: and is decrypting messages created with an algorithm suite including a signature algorithm,
    TEXT[!MUST]: any released plaintext MUST NOT be considered signed data until this operation finishes
    TEXT[!MUST]: successfully.
    TEXT[!MUST]: This means that callers that process such released plaintext MUST NOT consider any processing successful
    TEXT[!MUST]: until this operation completes successfully.
    TEXT[!MUST]: Additionally, if this operation fails, callers MUST discard the released plaintext and encryption context
    TEXT[!MUST]: and MUST rollback any processing done due to the released plaintext or encryption context.

SPECIFICATION: [Encrypt](specification/client-apis/encrypt.md)
  SECTION: [Input](#input)
    TEXT[!MUST]: - The input to the Encrypt operation MUST accept a required [plaintext](#plaintext) argument.
    TEXT[!MUST]: - The input to the Encrypt operation MUST accept a [cryptographic Materials Manager (CMM)](../framework/cmm-interface.md) and a [keyring](../framework/keyring-interface.md) argument.
    TEXT[!SHOULD]:   The keyring and CMM inputs SHOULD be optional.
    TEXT[!MUST]:   The Encrypt operation MUST validate that exactly one keyring or CMM was provided by the caller.
    TEXT[!MUST]:   If the caller does not provide exactly one of a keyring or CMM, the Encrypt operation MUST fail.
    TEXT[!MUST]: - The input to the Encrypt operation MUST accept an optional [Algorithm Suite](#algorithm-suite) argument.
    TEXT[!MUST]: - The input to the Encrypt operation MUST accept an optional [Encryption Context](#encryption-context) argument.
    TEXT[!MUST]: - The input to the Encrypt operation MUST accept an optional [Frame Length](#frame-length) argument.
    TEXT[!MAY]: If the [plaintext](#plaintext) is of unknown length, the caller MAY also input a
    TEXT[!MAY]: [Plaintext Length Bound](#plaintext-length-bound).
    TEXT[!SHOULD]: Implementations SHOULD ensure that a caller is not able to specify both a [plaintext](#plaintext)
    TEXT[!SHOULD]: with known length and a [Plaintext Length Bound](#plaintext-length-bound) by construction.
    TEXT[!MUST]: If a caller is able to specify both an input [plaintext](#plaintext) with known length and
    TEXT[!MUST]: a [Plaintext Length Bound](#plaintext-length-bound),
    TEXT[!MUST]: the [Plaintext Length Bound](#plaintext-length-bound) MUST NOT be used during the Encrypt operation
    TEXT[!MUST]: and MUST be ignored.

  SECTION: [Plaintext](#plaintext)
    TEXT[!MUST]: This MUST be a sequence of bytes.
    TEXT[!MAY]: This input MAY be [streamed](streaming.md) to this operation.
    TEXT[!SHOULD]: If an implementation requires holding the input entire plaintext in memory in order to perform this operation,
    TEXT[!SHOULD]: that implementation SHOULD NOT provide an API that allows this input to be streamed.

  SECTION: [Encryption Context](#encryption-context)
    TEXT[!MUST,implementation]: If the input encryption context contains any entries with a key beginning with `aws-crypto-`,
    TEXT[!MUST,implementation]: the encryption operation MUST fail.

  SECTION: [Algorithm Suite](#algorithm-suite)
    TEXT[!MUST,implementation]: The [algorithm suite](../framework/algorithm-suites.md) that MUST be used for encryption.
    TEXT[!MUST]: This algorithm suite MUST be [supported for the ESDK](../framework/algorithm-suites.md#supported-algorithm-suites-enum).

SPECIFICATION: [Streaming](specification/client-apis/streaming.md)
  SECTION: [Overview](#overview)
    TEXT[!MAY]: The AWS Encryption SDK MAY provide APIs that enable streamed [encryption](encrypt.md)
    TEXT[!MAY]: and [decryption](decrypt.md).
    TEXT[!SHOULD]: If an implementation requires holding the entire input in memory in order to perform the operation,
    TEXT[!SHOULD]: that implementation SHOULD NOT provide an API that allows the caller to stream the operation.
    TEXT[!SHOULD]: APIs that support streaming of the encrypt or decrypt operation SHOULD allow customers
    TEXT[!SHOULD]: to be able to process arbitrarily large inputs with a finite amount of working memory.

  SECTION: [Release](#release)
    TEXT[!MUST]: The decrypt and encrypt operations specify when output bytes MUST NOT be released
    TEXT[!MUST]: and when they SHOULD be released.

  SECTION: [Inputs](#inputs)
    TEXT[!MUST]: In order to support streaming, the operation MUST accept some input within a streaming framework.
    TEXT[!MUST]: - There MUST be a mechanism for input bytes to become consumable.
    TEXT[!MUST]: - There MUST be a mechanism to indicate that there are no more input bytes.

  SECTION: [Outputs](#outputs)
    TEXT[!MUST]: In order to support streaming, the operation MUST produce some output within a streaming framework.
    TEXT[!MUST]: - There MUST be a mechanism for output bytes to be released.
    TEXT[!MUST]: - There MUST be a mechanism to indicate that the entire output has been released.
    TEXT[!MUST]: Operations MUST NOT indicate completion or success until an end to the output has been indicated.

  SECTION: [Behavior](#behavior)
    TEXT[!MUST]: The behavior of the operation specifies how the operation processes consumable bytes,
    TEXT[!MUST]: and specifies when processed bytes MUST NOT and SHOULD be released.

SPECIFICATION: [Message Body AAD](specification/data-format/message-body-aad.md)
  SECTION: [Body AAD Content](#body-aad-content)
    TEXT[!MUST,implementation]: - [Non-framed data](message-body.md#non-framed-data) MUST use the value `AWSKMSEncryptionClient Single Block`.
    TEXT[!MUST,implementation]: - The [regular frames](message-body.md#regular-frame) in [framed data](message-body.md#framed-data) MUST use the value `AWSKMSEncryptionClient Frame`.
    TEXT[!MUST,implementation]: - The [final frame](message-body.md#final-frame) in [framed data](message-body.md#framed-data) MUST use the value `AWSKMSEncryptionClient Final Frame`.

  SECTION: [Sequence Number](#sequence-number)
    TEXT[!MUST,implementation]: For [framed data](message-body.md#framed-data), the value of this field MUST be the [frame sequence number](message-body.md#sequence-number).
    TEXT[!MUST,implementation]: For [non-framed data](message-body.md#non-framed-data), the value of this field MUST be `1`.

  SECTION: [Content Length](#content-length)
    TEXT[!MUST,implementation]: - For [non-framed data](message-body.md#non-framed-data), this value MUST equal the length, in bytes,
    TEXT[!MUST,implementation]:   of the plaintext data provided to the algorithm for encryption.
    TEXT[!MUST,implementation]: - For [framed data](message-body.md#framed-data), this value MUST equal the length, in bytes,
    TEXT[!MUST,implementation]:   of the plaintext being encrypted in this frame.
    TEXT[!MUST,implementation]:   - For [regular frames](message-body.md#regular-frame), this value MUST equal the value of
    TEXT[!MUST,implementation]:     the [frame length](message-header.md#frame-length) field in the message header.
    TEXT[!MUST,implementation]:   - For the [final frame](message-body.md#final-frame), this value MUST be greater than or equal to
    TEXT[!MUST,implementation]:     0 and less than or equal to the value of the [frame length](message-header.md#frame-length)
    TEXT[!MUST,implementation]:     field in the message header.

SPECIFICATION: [Message Body](specification/data-format/message-body.md)
  SECTION: [Non-Framed Data](#non-framed-data)
    TEXT[!MUST]: Non-framed data MUST be serialized as, in order,
    TEXT[!MUST]: IV,
    TEXT[!MUST]: Encrypted Content Length,
    TEXT[!MUST]: Encrypted Content,
    TEXT[!MUST]: and Authentication Tag.

  SECTION: [IV](#iv)
    TEXT[!MUST,implementation]: The IV MUST be a unique IV within the message.
    TEXT[!MUST]: The length of the serialized IV MUST be [IV Length](message-header.md#iv-length) bytes.
    TEXT[!MUST]: The IV MUST be interpreted as bytes.

  SECTION: [Encrypted Content Length](#encrypted-content-length)
    TEXT[!MUST,implementation]: The length MUST NOT be greater than `2^36 - 32`, or 64 gibibytes (64 GiB),
    TEXT[!MUST,implementation]: due to restrictions imposed by the [implemented algorithms](../framework/algorithm-suites.md).
    TEXT[!MUST]: The length of the serialized encrypted content length MUST be 8 bytes.
    TEXT[!MUST]: The encrypted content length MUST be interpreted as a Uint64.

  SECTION: [Encrypted Content](#encrypted-content)
    TEXT[!MUST]: The length of the serialized encrypted content MUST be equal to the value of the [Encrypted Content Length](#encrypted-content-length) field.
    TEXT[!MUST]: The encrypted content MUST be interpreted as bytes.

  SECTION: [Authentication Tag](#authentication-tag)
    TEXT[!MUST]: The length of the serialized authentication tag MUST be equal to the [authentication tag length](../framework/algorithm-suites.md#authentication-tag-length) of the [algorithm suite](../framework/algorithm-suites.md) specified by the [Algorithm Suite ID](message-header.md#algorithm-suite-id) field.
    TEXT[!MUST]: The authentication tag MUST be interpreted as bytes.

  SECTION: [Framed Data](#framed-data)
    TEXT[!MUST]: - The total bytes allowed in a single frame MUST be less than or equal to `2^32 - 1`.
    TEXT[!MUST,implementation]: - The number of frames in a single message MUST be less than or equal to `2^32 - 1`.

  SECTION: [Regular Frame](#regular-frame)
    TEXT[!MUST]: A regular frame MUST be serialized as, in order,
    TEXT[!MUST]: Sequence Number,
    TEXT[!MUST]: IV,
    TEXT[!MUST]: Encrypted Content,
    TEXT[!MUST]: and Authentication Tag.

  SECTION: [Sequence Number](#sequence-number)
    TEXT[!MUST,implementation]: Framed Data MUST start at Sequence Number 1.
    TEXT[!MUST,implementation]: Subsequent frames MUST be in order and MUST contain an increment of 1 from the previous frame.
    TEXT[!MUST]: The length of the serialized sequence number MUST be 4 bytes.
    TEXT[!MUST]: The sequence number MUST be interpreted as a UInt32.

  SECTION: [IV](#iv-1)
    TEXT[!MUST,implementation]: Each frame in the [Framed Data](#framed-data) MUST include an IV that is unique within the message.
    TEXT[!MUST,implementation]: The IV length MUST be equal to the IV length of the algorithm suite specified by the [Algorithm Suite ID](message-header.md#algorithm-suite-id) field.
    TEXT[!MUST]: The IV MUST be interpreted as bytes.

  SECTION: [Encrypted Content](#encrypted-content-1)
    TEXT[!MUST,implementation]: The length of the encrypted content of a Regular Frame MUST be equal to the Frame Length.
    TEXT[!MUST]: The encrypted content MUST be interpreted as bytes.

  SECTION: [Authentication Tag](#authentication-tag-1)
    TEXT[!MUST,implementation]: The authentication tag length MUST be equal to the authentication tag length of the algorithm suite
    TEXT[!MUST,implementation]: specified by the [Algorithm Suite ID](message-header.md#algorithm-suite-id) field.
    TEXT[!MUST]: The authentication tag MUST be interpreted as bytes.

  SECTION: [Final Frame](#final-frame)
    TEXT[!MUST,implementation]: Framed data MUST contain exactly one final frame.
    TEXT[!MUST,implementation]: The final frame MUST be the last frame.
    TEXT[!MUST,implementation]: The length of the plaintext to be encrypted in the Final Frame MUST be
    TEXT[!MUST,implementation]: greater than or equal to 0 and less than or equal to the [Frame Length](message-header.md#frame-length).
    TEXT[!SHOULD,implementation]: - When the length of the Plaintext is an exact multiple of the Frame Length
    TEXT[!SHOULD,implementation]:   (including if it is equal to the frame length),
    TEXT[!SHOULD,implementation]:   the Final Frame encrypted content length SHOULD be equal to the frame length but MAY be 0.
    TEXT[!MUST,implementation]: - When the length of the Plaintext is less than the Frame Length,
    TEXT[!MUST,implementation]:   the body MUST contain exactly one frame and that frame MUST be a Final Frame.
    TEXT[!MUST]: A final frame MUST be serialized as, in order,
    TEXT[!MUST]: Sequence Number End,
    TEXT[!MUST]: Sequence Number,
    TEXT[!MUST]: IV,
    TEXT[!MUST]: Encrypted Content Length,
    TEXT[!MUST]: Encrypted Content,
    TEXT[!MUST]: and Authentication Tag.

  SECTION: [Sequence Number End](#sequence-number-end)
    TEXT[!MUST,implementation]: The value MUST be encoded as the 4 bytes `FF FF FF FF` in hexadecimal notation.
    TEXT[!MUST]: The length of the serialized sequence number end MUST be 4 bytes.
    TEXT[!MUST]: The sequence number end MUST be interpreted as bytes.

  SECTION: [Sequence Number](#sequence-number-1)
    TEXT[!MUST,implementation]: The Final Frame Sequence number MUST be equal to the total number of frames in the Framed Data.
    TEXT[!MUST]: The length of the serialized sequence number MUST be 4 bytes.
    TEXT[!MUST]: The sequence number MUST be interpreted as a UInt32.

  SECTION: [IV](#iv-2)
    TEXT[!MUST,implementation]: The IV MUST be a unique IV within the message.
    TEXT[!MUST,implementation]: The IV length MUST be equal to the IV length of the [algorithm suite](../framework/algorithm-suites.md) that generated the message.
    TEXT[!MUST]: The IV MUST be interpreted as bytes.

  SECTION: [Encrypted Content Length](#encrypted-content-length-1)
    TEXT[!MUST]: The length of the serialized encrypted content length field MUST be 4 bytes.
    TEXT[!MUST]: The encrypted content length MUST be interpreted as a UInt32.

  SECTION: [Encrypted Content](#encrypted-content-2)
    TEXT[!MUST]: The length of the serialized encrypted content MUST be equal to the value of the [Encrypted Content Length](#encrypted-content-length-1) field.
    TEXT[!MUST]: The encrypted content MUST be interpreted as bytes.

  SECTION: [Authentication Tag](#authentication-tag-2)
    TEXT[!MUST,implementation]: The authentication tag length MUST be equal to the authentication tag length of the algorithm suite
    TEXT[!MUST,implementation]: specified by the [Algorithm Suite ID](message-header.md#algorithm-suite-id) field.
    TEXT[!MUST]: The authentication tag MUST be interpreted as bytes.

SPECIFICATION: [Footer Structure](specification/data-format/message-footer.md)
  SECTION: [Overview](#overview)
    TEXT[!MUST]: When an [algorithm suite](../framework/algorithm-suites.md) includes a [signature algorithm](../framework/algorithm-suites.md#signature-algorithm),

    TEXT[!MUST,implementation]: in the order of serialization.

    TEXT[!MUST,implementation]: [message footer](message-footer.md) serialized after the [message body](message-body.md).
    TEXT[!MUST]: If the algorithm suite does not contain a signature algorithm, the message MUST NOT  contain a message footer.
